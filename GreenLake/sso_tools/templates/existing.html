Troubleshoot SSO login

Diagnose a user's SAML assertion

Upload a capture from a failed (or successful) GreenLake login. The tool extracts the SAML response and runs nine checks against HPE GreenLake expectations: entity ID, ACS URL, certificates, NameID, and profile attributes.

How it works

  1. 1 Capture a login attempt with SAML-tracer (browser extension) or export a HAR from DevTools while the user signs in.
  2. 2 Upload the export on the right. Parsing stays on this server; nothing is sent elsewhere.
  3. 3 Review checks one at a time. Fail = fix in IdP config. Review = compare the value to the user's GreenLake SSO mapping.

Accepted files

  • .json SAML-tracer export
  • .har Browser network capture
  • .xml Raw SAML response or IdP metadata
  • .txt Base64 SAML or URL-encoded POST body

Plain server log lines without an embedded SAML payload are not supported. Use a tracer or HAR instead.
